# NIF Technical Operations Suite — Codebase Analysis

## 1. Project Summary
- Purpose: AI-driven operations management platform for HDPE & PVC pipe professionals (multi-tenant SaaS).
- Main stack: React + TypeScript + Vite + Tailwind; Supabase backend (Postgres, Auth, Edge Functions in Deno); AI via Lovable/Gemini.

## 2. Entry Points & Runtime
- HTML entry: [index.html](index.html)
- App entry: [src/main.tsx](src/main.tsx) → [src/App.tsx](src/App.tsx)
- Bundler: Vite (vite.config.ts) with SWC React plugin
- Dev server: IPv6 host ::, port 8080

## 3. Top-level Structure
- `src/` – frontend source: components, pages, hooks, contexts, integrations, lib
- `supabase/` – database migrations, Edge Functions, config.toml
- `public/` – PWA manifest and service worker
- Key files: package.json, tsconfig.json, vite.config.ts, tailwind.config.ts

## 4. Frontend Architecture
- Routing: `react-router-dom` with role-based dashboard routing in `src/pages/Dashboard.tsx`
- State: TanStack React Query for server state; `AuthContext` for auth/org membership; local state via hooks
- Forms: React Hook Form + Zod
- UI: shadcn-ui (Radix + Tailwind)
- Notable hooks: `useAiAssistant` (SSE streaming), `use-toast`, `use-mobile`

## 5. Backend & Integrations
- Platform: Supabase (Postgres + Auth + Storage + Edge Functions)
- Functions: Deno-based Edge Functions in `supabase/functions/` (ai-assistant, process-report, stock-analysis, etc.)
- AI integration: `supabase/functions/ai-assistant/index.ts` prefers Lovable gateway then Gemini; fallback to rule text
- Supabase config: `supabase/config.toml` enables `verify_jwt = true` for functions

## 6. Database (summary)
- Migrations: 28+ SQL files in `supabase/migrations/`
- Major tables: organizations, organization_memberships, profiles, clients, quotations, quotation_items, inventory, projects, field_reports, equipment, worker_claims, expenses, messages, ai_intelligence_logs
- RLS: Row Level Security used extensively; helper functions in DB (e.g., `has_org_role`, `is_maintenance_admin`) used in policies

## 7. Authentication & Authorization
- Auth provider: Supabase Auth (email/password)
- Client-side: `src/contexts/AuthContext.tsx` manages session, profile fetch, memberships, active role/org
- Authorization: RLS policies + helper DB functions; role-based access across frontend

## 8. API Endpoints & Usage Patterns
- Supabase REST (PostgREST) endpoints used via `@supabase/supabase-js`
- Edge functions called at `/functions/v1/{name}` (ai-assistant, process-report, etc.)
- Common patterns: React Query useQuery + useMutation wrapping Supabase calls

## 9. Config & Secrets
- Client env vars: `VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY`
- Server secrets expected in environment: `LOVABLE_API_KEY`, `GEMINI_API_KEY`, SUPABASE service role keys used server-side

## 10. Error Handling, Logging, Testing
- Error handling: try/catch in async functions; UI surfaces errors via toasts and error UI
- Logging: `console.error` used; DB audit via `log_audit_event()` triggers
- Testing: Vitest present but minimal tests (`src/test/example.test.ts`)

## 11. Security Observations
- Positives: RLS enforced, JWT verification for functions, rate-limiting middleware on AI functions
- Risks:
  - Potential prompt injection via AI functions (user-controlled `prompt` + `data` forwarded to AI)
  - LocalStorage session persistence exposes tokens to XSS; Content Security Policy not present in repo
  - Storage bucket ACLs and RLS should be audited (field_report_photos, claim attachments)
  - Loose TypeScript flags (`noImplicitAny: false`) reduce compile-time guarantees

## 12. Performance Observations
- Good: Vite + SWC, React Query caching, streaming AI via SSE
- Improvements: enforce list pagination, optimize client-side PDF generation, image optimization, component memoization

## 13. Code Quality
- Consistent naming conventions (PascalCase components, use* hooks)
- Areas to tighten: stricter TS config, reduce large page components (HR, AppSettings), centralize magic strings/constants

## 14. Recommended Refactors (high priority)
- Split large pages into focused subcomponents (`src/pages/HR.tsx`, `src/pages/AppSettings.tsx`)
- Move AI system prompts and retry logic out of `supabase/functions/ai-assistant/index.ts` to reusable modules
- Extract organization-specific context from `AuthContext` into `OrganizationContext`
- Add hooks wrapping common Supabase queries (useOrganizations, useProjects, useInventory)
